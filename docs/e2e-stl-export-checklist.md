# STL 导出 · 手动 E2E 测试清单（任务-07）

使用示例 GPX（项目内或自备户外轨迹），每种底图形状至少跑一组。

## 前置

- [ ] `npm run dev` 启动应用
- [ ] 导入 GPX，地图中心与轨迹对齐
- [ ] 预览区 2D/3D 切换正常

## 圆形底图

- [ ] 默认参数点击「生成并下载 STL」
- [ ] 保存对话框出现，选择路径后 ZIP 生成成功
- [ ] 解压 ZIP，含且仅含：`Terrain_Main.stl`、`Trail_Line.stl`、`Tray_Base.stl`（文件名精确匹配）
- [ ] 三个 STL 可在切片软件中打开，无破面报错

## 矩形底图

- [ ] 切换矩形，调整长宽
- [ ] 导出 ZIP，三文件齐全
- [ ] 可选：开启边框刻字后 `Tray_Base` 体积变化可见

## 正多边形底图

- [ ] 边数 3–8 各试一组（或抽样 6 边形）
- [ ] 导出成功

## 装配公差

- [ ] 默认 `trailToleranceMm=0.15`：`Trail_Line` 宽度 ≈ 轨迹宽度 − 0.3mm（相对预览轨迹略窄）
- [ ] 增大托盘公差后 `Tray_Base` 凹槽相对主模型外轮廓变大（切片截面或测量）

## 磁铁孔（可选）

- [ ] 关闭磁铁：三件模型无底面孔
- [ ] 仅冰箱贴孔：`Tray_Base` 底面有浅孔
- [ ] 拼接孔：托盘凹槽底 + 主模型底配对孔

## 错误与取消

- [ ] 未导入 GPX 点击生成 → 中文提示，不崩溃
- [ ] 托盘厚度 ≤ 凹槽深度 → 校验失败提示
- [ ] 保存对话框点取消 → 「已取消保存」，临时目录已清理

## 性能

- [ ] 典型 GPX（约 10MB 级源文件或数千点）生成时侧栏显示进度百分比，窗口可拖动不卡死
