## 目标

搭建 Electron + Vue 3 工程骨架，明确渲染进程与主进程职责边界，并建立 IPC 通信与参数状态管理的统一约定。

## 范围

### 包含

- 初始化 Electron 应用（主进程入口、窗口生命周期）
- Vue 3 渲染进程工程（Vite 或等价构建链）
- 主进程 / 渲染进程 IPC 通道设计（命名、类型、错误传递）
- 全局配置状态结构（与各模块参数对应的 TypeScript/JSON Schema 草案）
- 主进程占位：重度计算任务队列（避免阻塞 UI）
- 开发脚本：`dev` / `build` / `package` 可运行

### 不包含

- 具体 DEM/网格/STL 算法实现
- 地图组件与 3D 预览实现

## 验收标准

- [ ] 应用可启动，窗口展示 Vue 根组件
- [ ] 渲染进程可通过 IPC 调用主进程示例方法并收到响应
- [ ] 文档或代码注释中写明：UI 仅展示与状态管理；网格生成、STL 写出、ZIP 打包均在主进程
- [ ] 参数状态模型覆盖 PRD 五个功能模块的字段占位（可为默认值）

## 技术要点

- **渲染进程**：界面、地图交互、轻量 3D 预览
- **主进程**：GPX 解析、DEM 采样、布尔/挖槽、STL 生成、文件系统与 ZIP

## 交付物

- 可运行的 Electron 项目目录结构
- `ipc` 模块或等价抽象
- `shared/types`（或类似）中的配置类型定义
