# 物料区域 — 产品需求文档（MVP）

| 项目     | 说明              |
| -------- | ----------------- |
| 产品     | TrailPrint 3D     |
| 文档版本 | v0.3（MVP）       |
| 状态     | 待评审            |
| 关联能力 | 3D 预览、STL 下载 |

---

## 1. 背景与目标

用户完成地形导出后，可能需要 **配套外壳** 等物料用于保护与展示。本需求新增「物料区域」，MVP 提供三项能力：**浏览、预览、下载**。

**范围边界**：

- 物料 STL **独立下载**，不并入 `Terrain_Main / Trail_Line / Tray_Base` 主 ZIP。
- 物料 3D 模型由开发人员制作并放入资产目录；MVP 仅消费静态资产，不提供应用内生成或编辑。

**成功标准**：用户从应用首页进入「物料区域」，浏览官方物料列表，预览模型细节，下载所需 STL。

---

## 2. 设计决策

| 决策项   | 结论                       | 说明                                              |
| -------- | -------------------------- | ------------------------------------------------- |
| 模块入口 | 应用首页                   | 首页提供「物料区域」入口，点击进入物料浏览页      |
| 资产形态 | 预置 STL + `manifest.json` | 随应用打包；新增物料只需更新资产与清单            |
| 资产目录 | `resources/shells/`        | 含清单、STL、缩略图                               |
| 预览     | 复用现有 3D 预览能力       | 复用 `TerrainPreviewModal` / `TerrainMeshPreview` |
| 下载     | Electron 保存对话框        | 与现有导出一致：`dialog.showSaveDialog`           |
| 物料制作 | 不在 MVP UI 范围           | 由物料开发人员按 §5 约定产出资产                  |

| ------ | ---------------------------------------- | ------ |

## 4. 非功能需求

| 类别   | 要求                                                   |
| ------ | ------------------------------------------------------ |
| 性能   | 列表首屏渲染流畅；中等规模 STL 预览加载 ≤ 2s（目标值） |
| 离线   | 资产本地化，浏览 / 预览 / 下载全流程离线可用           |
| 一致性 | 遵循 [UI文档.md](./UI文档.md)，与现有应用视觉规范一致  |
| 可扩展 | 清单驱动列表渲染，新增物料无需改 UI 逻辑               |

## 5. 数据约定

### 5.1 目录结构

```
resources/shells/
├── manifest.json
├── {id}/
│   ├── model.stl
│   └── thumbnail.png
└── ...
```

### 5.2 清单格式

```ts
export interface ShellAsset {
  id: string;
  name: string; // 简体中文
  type: string; // 如「桌面」「壁挂」
  description?: string;
  thumbnail: string; // 相对路径
  stlPath: string; // 相对路径
  previewMeshPath?: string; // 可选，大文件预览加速
}

export interface ShellManifest {
  version: number;
  shells: ShellAsset[];
}
```

### 5.3 IPC 通道

| 通道             | 用途                    |
| ---------------- | ----------------------- |
| `SHELL_LIST`     | 读取 `manifest.json`    |
| `SHELL_LOAD`     | 加载 STL 用于 3D 预览   |
| `SHELL_DOWNLOAD` | 拷贝 STL 到用户指定路径 |

预览载荷可复用现有 mesh 类型（如 `TerrainMeshPayload`）。

## 6. 用户流程

```mermaid
flowchart LR
  A[首页] --> B[物料浏览页]
  B --> C[3D 预览]
  C --> D[下载 STL]
```

1. 用户在首页点击「物料区域」。
2. 进入物料浏览页，加载清单并展示卡片。
3. 点击卡片打开 3D 预览。
4. 确认后下载 STL 到本地。

---

## 7. UI 概要

### 7.1 首页

- 「物料区域」入口卡片或按钮
- 点击进入物料浏览页

### 7.2 物料浏览页

- 页头：标题「物料区域」、返回首页
- 主体：物料卡片列表（缩略图、名称、类型 / 描述）
- 空状态：占位文案与重试按钮

### 7.3 预览弹窗

- 顶部：物料名称、关闭
- 主区：3D 预览（旋转 / 缩放 / 平移）
- 底部：「下载 STL」；下载后可「在文件夹中显示」

---

## 8. 下载说明

| 项目   | 说明                                   |
| ------ | -------------------------------------- |
| 交付物 | 单个 `{物料名称}.stl`，单位 mm         |
| 独立性 | 与地形主 ZIP 分开下载                  |
| 命名   | 与清单 `name` 一致，去除非法文件名字符 |

---

## 9. 物料资产（开发侧）

物料 STL、缩略图及 `manifest.json` 由物料开发人员维护，按 §5 约定接入。MVP UI 假设资产已就绪。

可选增强：

- 提供 `previewMeshPath` 低精度网格以优化预览
- 产出前做 mesh 质量校验

---

## 10. 验收标准

1. 首页可进入「物料区域」，展示至少 1 条示例物料。
2. 点击卡片可 3D 预览并交互查看。
3. 可下载 STL 至用户指定路径，并「在文件夹中显示」。
4. 清单缺失或加载失败时有空状态与重试，应用不崩溃。
5. 物料下载独立于地形主 ZIP，全流程离线可用。
