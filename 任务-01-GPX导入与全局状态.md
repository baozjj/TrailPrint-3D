## 目标

实现 `.gpx` 文件上传与解析，将轨迹几何写入全局状态，供地图框选与后续模型生成使用。

## 范围

### 包含

- 文件选择 / 拖拽上传 `.gpx`
- 主进程解析 GPX（轨迹点序列、边界框）
- 渲染进程展示导入结果（点数、里程概览等轻量信息）
- 全局状态中保存：原始轨迹、边界框、地图中心建议值
- 与「GPX 轨迹优化」开关联动的数据管道接口（默认关闭，逻辑在任务-04 完善）

### 不包含

- 地图遮罩与平移缩放（任务-02）
- 轨迹宽度/厚度与挖槽（任务-04）

## 验收标准

- [ ] 用户可选择有效 `.gpx` 并成功解析
- [ ] 无效或空文件有明确中文错误提示
- [ ] 解析后轨迹可在后续任务中被地图模块读取（数据结构稳定）
- [ ] 大文件解析在主进程执行，UI 不出现长时间冻结

## 数据契约（建议）

```ts
interface GpxImportResult {
  points: Array<{ lat: number; lon: number; ele?: number }>;
  bounds: { minLat; maxLat; minLon; maxLon };
  trackName?: string;
}
```

## 交付物

- GPX 解析服务（主进程）
- 「导入 GPX」UI 入口（可与任务-08 合并验收）
- 全局 store 中的 `gpx` 切片
