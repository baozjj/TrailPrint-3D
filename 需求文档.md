# 产品需求文档 (PRD)

## 📌 1. 产品基本信息

- **产品名称（暂定）：** 「迹印 TrailPrint 3D」(英文名可选：TopoTrail 3D / TrailCrafter)
- **产品定位：** 一款面向户外爱好者与 3D 打印玩家的 Web 工具。将 GPX 徒步/骑行轨迹一键转化为可分色打印、免胶水磁吸拼装的「托盘式实体 3D 地形纪念碑」。
- **核心交付物：** 包含 3 个完美匹配的 STL 文件的 ZIP 压缩包（地形主模型、轨迹模型、带文字的托盘底座）。

## 🗺️ 2. 核心用户工作流 (User Journey)

1. **导入：** 用户上传 `.gpx` 文件。
2. **框选：** 选择底座形状与尺寸，在地图上平移/缩放，使轨迹落在最佳构图范围内。
3. **塑形：** 调节地形的夸张比例、平滑度，以及轨迹的粗细。
4. **定制：** 设置底座厚度、内陷深度，并在边框添加纪念文字。
5. **装配：** 配置 3D 打印公差，输入磁铁尺寸生成免胶水组装孔。
6. **导出：** 预览并下载生成的 STL 压缩包。

## ⚙️ 3. 功能模块详细需求

### 模块一：地图选取与尺寸定义 (Map & Cropping)

此模块决定了 3D 模型的外部轮廓（Bounding Box）和物理边界。

#### 1.1 形状选择器 (Shape Selector)

- 单选项：圆形 (Circle)、矩形 (Rectangle)、正多边形 (Regular Polygon)。

#### 1.2 物理尺寸设置 (Physical Dimensions)

- **联动逻辑：** 根据 1.1 的选择动态展示输入框。
  - 如果选【圆形】：展示「打印半径 (mm)」输入框。
  - 如果选【矩形】：展示「长度 (mm)」和「宽度 (mm)」输入框。
  - 如果选【正多边形】：展示「边数 (N)」下拉菜单（如 3-8 边），和「边长 (mm)」输入框。

> **注：** 此处输入的尺寸为最终底座的最外围尺寸。

#### 1.3 交互式地图视窗 (Interactive Map Mask)

- 在 UI 上展示一个底图（包含真实高程渲染的视觉图）。
- 根据 1.1 和 1.2 形成一个"半透明遮罩层 (Mask)"。
- 允许用户通过鼠标拖拽平移地图、滚轮缩放地图，确保 GPX 轨迹位于遮罩内的理想位置。

### 模块二：主模型生成 (Terrain Main Model)

此模块用于生成带底部厚度、表面有起伏的山体模型。

#### 2.1 模型基础厚度 (Base Solid Thickness)

- 输入框 (mm)。定义地形最低点到主模型纯平底面的实体厚度，确保山体模型打印时足够坚固。

#### 2.2 Z 轴高程夸张 (Z-Exaggeration)

- 滑动条 (1.0x - 5.0x)。用于放大真实海拔的高低落差。

#### 2.3 地形平滑度 (Terrain Smoothing)

- 单选档位：原始 / 轻度 / 中度 / 高度。用于过滤数字高程模型（DEM）带来的阶梯状噪点。

### 模块三：轨迹模型生成 (Trail Model)

此模块控制轨迹线条的形态，并在主模型上执行"挖槽"逻辑。

#### 3.1 GPX 轨迹优化 (GPX Simplify)

- 开关 (Toggle)。开启后自动过滤漂移点和毛刺。

#### 3.2 轨迹物理尺寸 (Trail Dimensions)

- **轨迹宽度 (Width)：** 输入框 (mm)。
- **轨迹厚度 (Depth)：** 输入框 (mm)。
- **核心逻辑关联：** 此厚度数值，即为主模型表面被向下挖槽的深度。

### 模块四：托盘式底座 (Tray Base)

此模块生成一个包裹主模型的托盘底座，用于承托主模型并提升成品观感。

#### 4.1 托盘结构参数 (Tray Structure)

- **总厚度 (Total Thickness)：** 输入框 (mm)。整个底座最高处的高度。
- **下陷深度 (Recess Depth)：** 输入框 (mm)。用于放置主模型的凹槽深度。（需做表单校验：下陷深度必须小于总厚度）。
- **边框宽度 (Rim Width)：** 输入框 (mm)。底座最外边缘到凹槽边缘的距离。

### 模块五：打印装配与磁铁防脱系统 (Assembly & Magnetics)

此模块是高级玩家的核心痛点，解决分件打印拼装不上的问题。

#### 5.1 3D 打印公差设置 (Tolerances)

- **轨迹装配公差：** 输入框 (默认 0.15mm)。主模型的轨迹凹槽需比轨迹实体模型大出此数值。
- **托盘装配公差：** 输入框 (默认 0.20mm)。底座的下陷凹槽需比主模型的外轮廓大出此数值。

#### 5.2 磁铁孔位生成器 (Magnet Holes)

- **启用开关：** 开启/关闭。
- **磁铁规格：** 直径 (mm) 和 厚度 (mm)。
- **孔位一 (冰箱贴孔)：** 复选框。勾选后，在底座最底面生成浅孔，用于吸附铁质表面。
- **孔位二 (无胶水拼接孔)：** 复选框。勾选后，在"底座凹槽的底面"和"主模型的底面"的绝对对应位置，生成匹配的孔洞。

## 📦 4. 交付物标准定义

用户点击"生成"后，系统需返回一个 `.zip` 压缩包，文件结构如下：

| 文件名             | 说明                                                                   |
| ------------------ | ---------------------------------------------------------------------- |
| `Terrain_Main.stl` | 底部纯平、带有固定厚度、表面已抠除轨迹槽、底部带（可选）磁铁孔的山体。 |
| `Trail_Line.stl`   | 纯轨迹线条，根据配置自动应用了缩小公差。                               |
| `Tray_Base.stl`    | 带有边框宽度、下陷凹槽、边缘文字、底部与内测带（可选）磁铁孔的底座。   |

## 5 整体架构选型

- **核心框架：** Electron + Vue 3。
- **职责划分：**
  - **渲染进程 (UI 层 - Vue 3)：** 仅负责界面展示、参数状态管理、地图交互与轻量级的 3D 模型结果预览。
  - **主进程 (核心逻辑层 - Node.js)：** 负责所有重度计算与系统级操作，确保界面永不卡顿。
