## 目标

用户点击「生成」后，主进程输出包含三个 STL 的 ZIP，文件名与结构符合 PRD，并触发本地下载。

## 交付物标准

| 文件名 | 说明 |
| --- | --- |
| `Terrain_Main.stl` | 底面纯平、固定厚度、表面已抠轨迹槽、可选底部磁铁孔 |
| `Trail_Line.stl` | 纯轨迹线条，已应用轨迹装配公差缩小 |
| `Tray_Base.stl` | 边框、下陷凹槽、边缘文字、底面/内侧可选磁铁孔 |

## 范围

### 包含

- 三网格 → 二进制 STL 写出（主进程）
- ZIP 打包与保存对话框 / 下载路径
- 生成过程进度与错误处理（中文）
- 「生成并下载 STL」按钮端到端联调

### 不包含

- 云端存储或账户体系

## 验收标准

- [ ] 单次生成 ZIP 内含上述三个文件，命名精确匹配
- [ ] `Trail_Line` 尺寸相对主模型凹槽满足任务-06 公差
- [ ] 主模型可单独放入托盘凹槽（满足托盘公差）
- [ ] 生成失败时 UI 展示可理解原因，不崩溃
- [ ] 10MB 级典型模型生成时 UI 保持响应（异步 + 进度）

## 技术要点

- STL 写出库选型（主进程）
- 临时目录清理
- IPC：`generateModels(config) → { zipPath }` 或流式传输

## 交付物

- 导出服务模块
- E2E 手动测试清单（示例 GPX + 各形状各一组）
